Forensic Lab setup with REMnux + Sift + FlareVM

This tutorial will guide you through the setup of a Forensic lab using:

  • VirtualBox 7
  • REMnux + Sift on Ubuntu 20.04
  • FlareVM on Windows 10

It is mostly a summary of notes I took while building my own lab.

Disclaimer

Do not follow this guide blindly!

Be very careful during the setup of your forensic lab, especially if you intend to manipulate and/or detonate malware. I am in no way responsible for any harm or damage you may cause and you should always make sure you are operating in a safe environment.

If you are not sure about what you’re doing, please stop and seek advice from professionals.

0x00 - Get an hypervisor

Although his setup will be using VirtualBox 7, it can be adapted to use another hypervisor.

0x01 - Linux box setup

This guide will start from an Ubuntu image but you may also start from the REMnux of Sift VM appliance and install the other REMnux/Sift on top of it.

Prepare Ubuntu 20.04

At the time of writing, REMnux does not support Ubuntu 22.04, so I will be using Ubuntu 20.04. Also, REMnux won’t run on ARM processors.

  1. Download the Ubuntu 20.04 image and create the VM from it.
  2. Proceed to the installation as you would for a bare metal installation.
  3. When it’s completed, click on “Device” in the VirtualBox menu and “Insert Guest Additions CD image…” and run it. It will fix the resolution and “optimize the guest operating system for better performance and usability”. Check the documentation for further details.
  4. Once ready, spin up a terminal and add yourself to the sudoers.

You should now have a clean Ubuntu setup on which you will be able to install REMnux and Sift.

0x03 - Network setup

‚ùó This part is very important if you want to isolate your lab and avoid infecting your host.

  1. Create a new network on VirtualBox
  2. Enable DHCP server
  3. Setup an IP range you can, in no way, confuse with your local network, e.g. 10.0.0.0/24 if your local network is 192.168.0.0/24
  4. Update the DHCP server address, mask, and lower/upper addresses accordingly
  5. Change the network adapter of the VMs to the one you just created
  6. Double check that there are no other active network adapters!

INetSim configuration (Linux box)

Configuring INetSim will allow to simulate “common internet services in a lab environment, e.g. for analyzing the network behaviour of unknown malware samples”.

Edit /etc/inetsim/inetsim.conf:

  • uncomment start_service dns
  • set service_bind_address to 0.0.0.0
  • set dns_default_ip to the IP of the box

Update DNS server address (Windows box)

Set the “Preferred DNS server” to the IP of the Linux box.
